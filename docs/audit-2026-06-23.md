# Audit 2026-06-23: distill, validate, reconcile

A distillation pass over the whole repository at HEAD `c5338e1` (post BL-026 /
ADR 0084), with three goals: distill what the code actually is, re-validate the
load-bearing external claims against trusted sources, and reconcile the
code-versus-documentation drift the two prior passes (2026-06-15, 2026-06-16)
had accrued. The cadence follows ADR 0023; this document is the point-in-time
record and the findings were fixed on the same branch rather than only logged.

## Scope and method

The distillation mapped every layer (spine, the fourteen subsystem physics
modules, the eleven estimators, the six interop adapters, the self-model, the
state and comms machinery, scenarios, and the OAuth issuer) and cross-checked it
against `STATUS.md`, `CLAUDE.md`, `LIMITATIONS.md`, and the backlog. The
validation re-derived the simulator's physics constants in a scratch script and
checked the standards and library citations against their canonical sources: the
Anthropic model identifiers and prompt-cache discipline through the `claude-api`
skill, the propagation and sensor physics against the published formulae, and
the DTN, OAuth, and KLV standards against the RFC and specification text.

## External validation (all confirmed)

The physics constants reproduce their literature values exactly: the Friis
free-space constant `20 log10(4 pi / c) = -147.55 dB`, the Johnson-Nyquist noise
floor `kTB = -173.975 dBm/Hz` at 290 K (the rounded `-174` the code refines),
the Koschmieder visibility constant `-ln(0.02) = 3.912`, the ITU-R P.526
knife-edge `J(0) = 6.0 dB`, the Rician fade's unit-mean power for every
K-factor, the methanol LHV `19.9 MJ/kg = 5.53 Wh/g` and the EFOY `0.9 L/kWh ->
1.4 Wh/g` electrical figure at a derived 25 percent system efficiency, and the
Johnson DRI cycle ratios (detection 1, recognition 3, identification 6). The
Anthropic layer is current: the default `claude-haiku-4-5-20251001` and advanced
`claude-sonnet-4-6` identifiers are valid, the adaptive-thinking capability set
(Opus 4.6 / 4.7 / 4.8 and Sonnet 4.6, with Haiku 4.5 correctly excluded) matches
the API, and the `cache_control: {"type": "ephemeral"}` shape with its five-minute
TTL is the documented prompt-cache discipline. The standards citations hold:
BPv7 is RFC 9171 (adapted from RFC 5050, BPv6), BPSec is RFC 9172, and the OAuth
2.0 Security BCP is RFC 9700.

The one external-citation defect: the OAuth refresh-token family-revocation
guidance is RFC 9700 **section 4.14**, not 4.13. The 4.13 in `auth/oauth.py` is
the section number from `draft-ietf-oauth-security-topics`, which the published
RFC renumbered. Corrected in both call sites; comment-only, no behaviour change.

## Dispositioned (no action)

The `BL-031a` identifier follows the existing `BL-005a` / `BL-005b` convention
and is not a defect. `BL-006` ("full profile field model") is correctly
`[in-progress]`: `ProfileModel` is still the minimal `extra="allow"` gate that
validates only `name` and the safety sections, not the full field schema. The
active position EKF is already documented by
`docs/model-cards/estimator-position-kalman.md` (whose title is "Position
(nonlinear EKF, GNSS/INS fusion)"), so no new card is required. The
`inference_request` entry in `policy.py` is a dispositioned forward-classification
(ADR 0033, allow-listed in the tier-coverage test), not drift. The open BL-111
inference-status basis question remains an intentional design decision recorded
at the 2026-06-16 audit, out of scope here.

## Posture

The simulator is approximately where `STATUS.md` now says it is. No
privilege-escalation, audit-bypass, or physics-fidelity defects surfaced; the
findings were drift between a fast-moving code surface and its documentation, plus
one stale estimator-read omission. The validation against trusted sources came
back clean on every load-bearing physics, model, and standards claim.
