# AUDIT 2026-06-06

A full-repository validation audit (not a delta against a single prior
baseline). Conducted per the [ADR 0023](adr/0023-audit-cadence-and-regression-suite.md)
cadence on trigger (a), a sustained evolution of the spine since the
2026-06-01 pass ([`audit-2026-06-01.md`](audit-2026-06-01.md)): the ADR
0027/0028/0029/0030 auto-safing and FSM-completeness train, and the model-card
programme (PRs #101 through #105).

Conducted: 2026-06-06.
Branch audited: `claude/lucid-newton-XZBo5`, on `main`.
Engagement: a full validation sweep (documentation in depth, standards / NATO /
BOM verification against external sources, a live-VM probe, and the
lint / type / test / policy gates), plus the fixes this audit landed.

## 1. Executive summary

The codebase is healthy and, unusually, honest about itself. `ruff` is clean,
`mypy --strict` is clean across 146 source files, and 736 pytest tests were
green at the start of the pass (738 after this engagement). The interop
adapters are accurately implemented and carefully hedged ("none certified",
"shape-only"); the bill of materials is well-sourced against real vendor and
NATO references; and the live reference VM is genuinely running, ticking at
process scope, with `audit.degraded:false` and an arithmetically-correct
self-model. The findings below are matters of precision, documentation
freshness, and one live-surface gap, not correctness defects.

This engagement lands the unambiguous corrections plus the two judgment-call
items approved at review: the position estimator is relabelled from "EKF" to
the linear Kalman filter it actually is, and a first-class `state_transition`
control tool is registered behind [ADR 0031](adr/0031-state-transition-control-tool.md)
so a controller can drive the mission-posture FSM directly instead of only
through `scenario_inject`.

## 2. Method

The audit built and exercised the repository rather than reading it alone:
`uv sync`, `make check` (`ruff` + `mypy --strict` + `pytest`), `make policy`
(the em-dash and private-repo greps), and `make docs-build` (`mkdocs build
--strict`). It probed the live VM through the mounted MCP surface
(`device_info`, `device_health`, `self_model_assess`, `self_estimator_status`).
And it verified the externally-checkable claims against their sources: the
STANAG, CoT, MISB, NMEA, OGC, and SOSA standards, and the BOM rows.

Behaviour-preserving (the code read nested-first). |
| E | Low | STATUS cited "645 tests pass at HEAD" (and 365 / 580 elsewhere); the actual collected count was 736. | **Fixed.** Corrected to 738 with the history recorded; the `645` was a stale intermediate that never matched reality. |
| F | Low | 15 tools are tier-classified in `policy.py` but unregistered (`comms_publish`, `inference_cloud`, `db_reset`, `state_force_fault`, ...). Forward-classification per ADR 0007, but only `inference_cloud` was documented as such. | **Noted.** Intentional (the `test_tool_tier_coverage` test pins the registered-to-classified direction only). `state_transition` is removed from this set by finding A; the rest stay forward-looking. |

Minor, all documented in the code or conformance pages and left as-is: MISB
omits the mandatory ST 0601 Tag 1 checksum (a strict parser would reject, but
the posture is "shape-only"); NMEA hardcodes geoid separation to `0.0`
(HAE = MSL); STANAG 4074 is strictly the starter/slave connector, used here as
a 28 V charging bus (documented as an accessory-port draw); and the live
`device_info` reports `policy:"open"`, which is acceptable behind the OAuth
single-client lockdown and CIDR gate but worth a deployment-review note.

## 6. Live-VM probe

The mounted MCP surface answered. `device_info`: version 0.1.0, profile
`jetson-agx-orin`, transport HTTP, `tick_hz` 2.0, `audit.degraded:false`,
`anchor_degraded:false`, safety constraints SC-2 / SC-8 registered, zero
violations. `device_health`: tick 937 at `ts_s` 468.5 (engine ticking at
process scope, so the ADR 0024 fix holds), SoC 99.8 %, junction 30.9 C. The
`self_model_assess` numbers are arithmetically correct: endurance p50 3521 min
is 588 Wh / ~10 W * 60; thermal headroom 54.1 C is 85 - 30.88; inference
capacity 190 tok/s is 200 * 0.95. Estimator covariances are converging. The
single observation, finding A, was the mode pinned at `boot` with no registered
way to advance it; that is what ADR 0031 closes.

## 7. Residual recommendations

These are noted, not landed, and belong in the backlog rather than this PR:

- Rationalise the 15 classified-but-unregistered tools (finding F): either
  register the near-term ones (`tick_advance`, the scenario-control verbs) or
  annotate the frozensets in `policy.py` as forward-classifications so the
  intent is legible without cross-referencing the registry.
- If a real downstream MISB parser is ever targeted, emit the ST 0601 Tag 1
  checksum as the trailing item; today's "shape-only" posture is correct for
  the simulator but a strict consumer will reject a set without it.
- Record the live `policy:"open"` choice in the deployment guide's posture
  section, paired with the OAuth single-client and CIDR gate that make it
  acceptable.
