## 1. Executive summary

Eleven open findings carried forward from the 2026-05-24 baseline
close in this engagement, plus three new supply-chain findings
surfaced during the Phase 0 inventory and addressed in the same
stack. Every closure is pinned in
``tests/regression/test_audit_findings.py`` per ADR 0023.

The repository is in noticeably stronger shape than the 2026-05-24
baseline. The spine (audit, runner, state machine, OAuth provider)
now carries dedicated unit tests covering the full execution
contract; the OAuth provider arbitrates concurrent
read-modify-write sequences under an asyncio lock, persists file
state at mode ``0o600``, fsyncs parent directories after every
atomic rename, and revokes refresh-token families on detected
reuse per OAuth 2.1 BCP Â§4.13. The auto-update loop captures a
rollback marker after every successful deploy and refuses to
re-attempt a known-broken commit. CI grew a supply-chain job
(``pip-audit --strict`` plus ``bandit -r src/nous``) and the docs
build now emits a CycloneDX-JSON SBOM as a release artefact. The
lockfile is tracked. Every GitHub Action is SHA-pinned.

Quality gates remain green: ruff clean, mypy strict clean across
60 source files, **517 pytest tests pass** (up from 442 at the
start of the engagement and 365 at the 2026-05-24 baseline). The
new ``supply-chain`` CI job is green (``pip-audit --strict`` zero
vulnerabilities; ``bandit`` zero issues with four well-documented
``# nosec`` suppressions). ``mkdocs build --strict`` clean.
``make policy`` clean.

The audit found no privilege-escalation, no remote-code-execution,
no audit-bypass classes of bug, and no new critical findings. Four
open items carry forward (H2 mypy-strict for tests, N3 / N4 / N6 /
N7 polish items, N2 server-side live-VM probe, and the N8 ADR
0019-0022 implementation programme).

## 4. New observations from this engagement

### N16 (Low). Hypothesis examples database grows over time

``.hypothesis/examples/`` and ``.hypothesis/constants/`` accumulate
across runs. The Hypothesis-driven property tests already share an
examples database, so a future failing example surfaces sooner. The
test_policy_fuzz drift was caught precisely because the database
had grown to cover ``anthropic_cap_status``. Recommend documenting
the directory under the contributor runbook so a contributor with
a hard-to-reproduce flake can clear it locally without confusion.

### N17 (Low). Bandit suppressions sit inline; not centrally documented

The four ``# nosec`` annotations landed in this engagement
(``cot.py`` B405 / B406 / B314 and ``oauth.py`` B106) are
defensible individually, but a reviewer reading the CI output has
to chase down each annotation and its justification. A short
``docs/security/bandit-suppressions.md`` (or a section in
SECURITY.md) listing every suppression with its rationale would
make the SAST posture surveyable in one view. Low priority.

attributes, MISB ST 0601 BER length handling, OGC SensorThings

encode-only JSON envelope shaper, not a certified MQTT client;

explicitly cross-referencing ``LIMITATIONS.md`` L6 (parametric,
not medical-grade).

## 7. Recommended remediation order

The 2026-05-24 Â§7 order is mostly retired by this engagement. The
remaining open items, in priority order:

1. **N2 audit sink triage.** Server-side. Still the next live-VM
   action item.
2. **H2 mypy strict for the test tree.** ``pyproject.toml`` mypy
   ``files = ["src/nous"]`` extension to include ``tests/``. The
   test suite will need fixture-decorator relaxations; expect a
   one-PR mechanical pass with a few targeted ignores.
3. **N3 / N4 polish.** Richer ``state_get`` payload; snapshot /
   MCP parity test.
4. **N7 MISB KLV decode round-trip.** UTF-8-with-hex-fallback or
   explicit conformance-doc note.
5. **N17 bandit-suppressions doc.** A short security doc indexing
   every ``# nosec`` annotation.
6. **N8 ADR 0019-0022 implementation programme.** Multi-PR
   programme, each governed by the matching ADR.

Items 1 to 5 are opportunistic; item 6 is design work the project
has already committed to via the Accepted ADRs.
