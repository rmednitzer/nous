- BL-048 [done] (L3) Propagation-aware comms model (ADR 0053). A link with a `propagation` block now solves its RSSI, packet loss, and SNR-derived capacity each tick from a first-order link budget rather than holding the profile's static nominal: a Friis free-space path loss over the slant range from the device position (a lazy `position_fn` seam mirroring the `rng=` injection) to the link's peer, a constant excess-loss margin standing in for terrain and obstruction, and a log-normal shadowing draw from the engine RNG (ADR 0019). A new `src/nous/subsystems/propagation.py` holds the pure link-budget functions; the result feeds the existing `rssi_dbm` / `loss_pct` live fields, so the unchanged observation to particle-filter to `derive` to FSM pipeline degrades on its own. The model is additive and inert without config: every new channel (`capacity_bps`, `LinkEstimate.bandwidth_bps` / `capacity_bps`, the observation capacity, the outbox RNG) carries a legacy fallback, and a static link's `capacity_bps` is its rated `bandwidth_bps`, so all existing comms / estimator / outbox / scenario tests stay green. The four downstream couplings the earlier ADRs deferred all land here: `tx` caps the achieved rate at `capacity_bps` (supersedes the ADR 0051 bandwidth cap, realizes the ADR 0020 throughput-monotone-in-SNR invariant); `derive` gates health on a per-link fraction of each link's own bandwidth rather than the flat 5000 bps (closes the ADR 0051 trigger); the comms particle filter uses the modeled capacity as its expected throughput so an observed rate far below capacity weighs against the connected hypothesis (the scale sensitivity ADR 0051 recorded as absent); and the store-and-forward flush models per-link Bernoulli packet loss on a propagation link (amends ADR 0047). Demonstrated by `profiles/propagation-demo.yaml` and `tests/integration/test_propagation_demo.py` (drive east, watch the relay degrade CONNECTED to DENIED and recover), with unit coverage in `tests/unit/test_propagation.py` (the physics plus an SNR-monotone property), additions to `tests/unit/test_comms_subsystem.py` / `test_comms_estimator.py` / `test_comms_outbox.py`, and the strengthened SNR-monotone invariant in `tests/unit/test_subsystem_invariants.py`. The forced-state override and the `inject_comms_loss` / `set_link_state` seam still hard-override the physics.
- BL-088 [done] (L3) Higher-fidelity propagation (ADR 0054), the first increment past BL-048 / ADR 0053. Five additive, opt-in upgrades to `subsystems/propagation.py`, each defaulting to reproduce the ADR 0053 free-space budget so an un-opted link is byte-for-byte unchanged: (1) a log-distance path-loss exponent (`PL = FSPL(1 m) + 10 n log10 d`, `n = 2` free space, 2.7 to 3.5 urban, 4+ obstructed), the largest realism gain; (2) a single knife-edge diffraction loss (ITU-R P.526 `J(v)`) from a configured obstruction's height above the line of sight, the terrain-blockage case without a DEM; (3) a kTB thermal-noise floor (`-174 dBm/Hz + 10 log10 B + NF`) when a channel bandwidth is given, replacing the per-link constant; (4) a directional antenna pattern (parabolic-in-dB main lobe, `-3 dB` at the half-beamwidth, floored at the back-lobe) keyed on the bearing to the peer; (5) a Rician multipath fast-fade draw (K-factor in dB, Rayleigh at `K = 0`) on top of the log-normal shadowing, both from the engine RNG (ADR 0019). `solve_link_budget` gains one optional `fast_fade_db` argument; the other four read from `LinkPropagation`. `subsystems/comms.py` draws the fade in `_apply_propagation` and is otherwise unchanged, so the observation to filter to `derive` to FSM pipeline is untouched and the existing suite stays green. Covered by `tests/unit/test_propagation.py` (each function plus the reduces-to-ADR-0053-at-defaults checks) and `tests/integration/test_propagation_demo.py` (an obstruction degrading the link through the engine, multipath adding tick-to-tick variance). Remaining higher-fidelity work is split out: DEM-driven multi-obstacle terrain is BL-089; mesh / multi-hop routing stays in the BL-056 DTN layer.

- BL-098 [done] (L3) DTN per-node store cap (AUDIT-2026-06-15 H-1). `DtnNode.store` is unbounded: `originate()` accepts `lifetime_s=0` (expiry `None`) and `step()` leaves a bundle whose `dest_eid` names no node in the topology in its holder's store, with no route and no expiry, so sustained origination to an unreachable peer grows memory without bound and without a controller-visible signal. Done (ADR 0068): a configurable `dtn.max_store` (default 256) bounds each node's store; the `_admit` seam sheds the triage-worst held bundle (lowest precedence, newest) over the cap and counts it as `dropped`, so unroutable or no-expiry bundles cannot grow without bound and a low-precedence flood cannot evict a higher-precedence custody bundle. Covered by `test_store_cap_sheds_bundles_over_capacity` and `test_store_cap_keeps_higher_precedence` in `tests/unit/test_dtn_mesh.py`.
- BL-099 [done] (L3) DTN per-bundle retransmit cap (AUDIT-2026-06-15 H-2). The audit flagged custody retransmit as O(custody_retries^N) over an N-hop high-loss path. Resolved (ADR 0068): re-assessed as over-stated. - BL-100 [done] (L3) DTN restore-loss accounting (AUDIT-2026-06-15 H-3). On `restore()`, a custodial bundle held by a node EID absent from the rebuilt topology (a peer removed between restarts) is silently skipped with no counter, log, or `degraded` flag, so a custody guarantee breaks invisibly and the controller cannot tell delivered from discarded. Done (ADR 0068): `restore()` counts every bundle it skips for an absent holder node into a runtime `restore_lost` counter surfaced in the `dtn_mesh` read's counters, so the custody loss is visible rather than silent. - BL-101 [done] (L3) DtnStore distinguish load from save failures (AUDIT-2026-06-15 M-1). `DtnStore.load()` incremented `save_failures` on a read or decode error, so a corrupt-JSON restore reported `degraded=True` with a save-shaped `last_error`, and a controller could not tell a read fault from a write fault. Done: `load()` now increments a runtime `load_failures` and records `last_load_error` (the exception class only); `degraded` is true on either counter, and `status()` surfaces both pairs additively (ADR 0007), so a corrupt restore reads as a read fault with the save counters untouched. - BL-108 [done] (L3) Attribute the cause of a held or dropped package (AUDIT-2026-06-15 L-3). The outbox `attempts` counter and the DTN `dropped_total` each conflated distinct causes: an EMCON denial, a zero `capacity_bps`, and a propagation-loss draw all bumped `attempts`; max-hops, forward-loss, retry-exhaustion, and a store-cap eviction all bumped `dropped_total`. Done (ADR 0070): the comms `tx()` records a `last_tx_reason` on each `Link` (`sent` / `forced_down` / `emcon` / `no_capacity` / `empty`) without changing its integer return; the outbox tallies a cumulative `defer_causes` map (`link_down`, `loss`, and the link's `emcon` or `no_capacity`), counting a deferral once per link per flush and not counting a budget hold; and the mesh splits `dropped_total` into a `drop_causes` map (`max_hops`, `forward_loss`, `retry_exhausted`, `store_overflow`). The planned "drop counters in the snapshot" shipped as a runtime-only breakdown (the aggregate `dropped_total` stays persisted, the split restarts per process), avoiding a schema migration for no operational gain, consistent with the BL-100 `restore_lost` and BL-101 load-failure fields.
