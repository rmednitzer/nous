## 4. New observations from this engagement

### N18 (Low). ``CommsParticleFilter`` still accepts a fallback ``seed``

The constructor signature is now ``CommsParticleFilter(*, seed:
int = 0, rng: np.random.Generator | None = None)``. When ``rng``
is provided the seed is ignored, but the ``seed`` parameter
remains for backward compatibility with direct test instantiations.
A future tightening would deprecate the ``seed`` kwarg in favour
of "always pass an explicit RNG"; today the dual signature is
intentional to keep ``tests/unit/test_comms_estimator.py``
constructions terse.

## 6. Recommended remediation order

What remains, in priority:

1. **N2 audit sink triage.** Server-side. Same as the prior audit's
   open item.
2. **ADR 0021 implementation.** Per-subsystem tool modules under
   ``src/nous/tools/``. Pure refactor; medium-large PR scoped per
   module group.
3. **ADR 0022 implementation.** ``src/nous/safety/enforcer.py``
   plus wiring each STPA constraint through it. New module;
   medium-sized PR per constraint family.
4. **ADR 0019 / 0020 follow-ups.** Per-subsystem ``rng`` kwargs
   (no subsystem samples noise today, so the seam can wait);
   ``numpy.random`` grep-ban in source; tick-loop clock-source
   switch (currently on ``anyio.current_time`` for event-loop
   integration). Additional ADR 0020 invariants (full Peukert,
   SNR-throughput, position EKF covariance growth) under the
   established pattern.
