# Conformance posture: NMEA 0183

**Adapter:** `src/nous/interop/nmea0183.py` (BL-033)

**Standard:** NMEA 0183 v4.10. The simulator emits the
`$GPGGA` sentence (essential GPS fix data).

**v0.1 posture:** Encode produces a `$GPGGA` sentence with a computed
XOR checksum. Decode is a stub that delegates to `pynmea2` in L2.

**What is supported:** `$GPGGA` encode with lat, lon, and timestamp.
The encoded string carries the trailing `\r\n` per spec.

**What is omitted in v0.1:** Other sentences (`$GPRMC`, `$GPGSA`,
`$GPGSV`), multi-talker prefixes, decode (via pynmea2).

**Conformance claim:** None. The encode is byte-compatible with a
`$GPGGA`-aware consumer; broader conformance lands with BL-033.
